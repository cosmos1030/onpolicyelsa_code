ELSA 구현 수식 설명

코드 기반: lib/gkd_admm_trainer.py, lib/optimizers.py

1. 최적화 문제

원래 목표: task loss를 최소화하면서 sparsity 제약을 만족하는 $w$ 찾기.

$$\min_{w} \; \mathcal{L}(w) \quad \text{s.t.} \quad w \in \mathcal{S}_s$$

$\mathcal{S}_s$: magnitude 상위 $s$%만 살리고 나머지를 0으로 만드는 집합 (unstructured sparsity).

비볼록 제약을 다루기 위해 ADMM으로 변수 분리:

$$\min_{w,z} \; \mathcal{L}(w) \quad \text{s.t.} \quad w = z, \quad z \in \mathcal{S}_s$$

Augmented Lagrangian (scaled dual variable $u$ 형태):

$$\mathcal{L}_\lambda(w, z, u) = \mathcal{L}(w) + \frac{\lambda}{2}\|w - z + u\|^2$$

2. ADMM 업데이트 규칙

2-1. w-update (매 step)

Adam + proximal term. _proximal_update()에서 gradient에 직접 더해준 뒤 Adam step.

$$g_t \;\leftarrow\; \nabla_w \mathcal{L}(w_t) + \lambda\,(w_t - z + u)$$ $$w_{t+1} \;\leftarrow\; w_t - \text{Adam}(g_t)$$

코드 대응:

penalty = w.detach() - split.detach() + dual.detach()   # w - z + u
prox    = lmda * penalty
w.grad.add_(prox)                                        # gradient에 더함
super().step()                                           # Adam step

2-2. z-update (매 interval=32 steps)

Projection: $w + u$에서 magnitude 기준 상위 $(1-s)$% 살리고 나머지 zeroing.

$$z^{k+1} \;=\; \text{Proj}_{\mathcal{S}_s}(w + u)$$

코드 대응:

z_in  = w.detach() + dual.detach()          # w + u
z_new = projection([z_in], sparsity, ...)

2-3. u-update (z-update와 동시)

Dual ascent: residual $w - z$를 누적.

$$u^{k+1} \;=\; u^k + (w - z^{k+1})$$

코드 대응:

u_new = dual.detach() + (w.detach() - z_new)
dual.copy_(u_new)

2-4. Lambda 스케줄 (cosine)

$$\lambda_t \;=\; \lambda_0 + (\lambda_T - \lambda_0)\cdot\frac{1 - \cos(\pi\, t/T)}{2}$$

현재 설정: init_lmda=5e-5, final_lmda=5e-5 (상수) → 사실상 constant schedule.

3. Task Loss $\mathcal{L}(w)$

3-1. On-policy 생성

매 training step에서 현재 student $\pi_\theta$로 응답 샘플링:

$$y \;\sim\; \pi_\theta(\cdot \mid x), \quad y = (y_1, \ldots, y_T)$$

3-2. KD Loss (Reverse KL, top-$k$ 필터링)

$$\mathcal{L}_\text{KD} \;=\; \mathbb{E}_{x,y}\!\left[\frac{1}{T}\sum_{t=1}^{T} \sum_{v \in \mathcal{V}_k(t)} p_s^{(t)}(v)\,\log\frac{p_s^{(t)}(v)}{p_T^{(t)}(v)}\right]$$
  • $p_s^{(t)}$: student의 위치 $t$ 분포
  • $p_T^{(t)}$: teacher (dense 원본 모델)의 위치 $t$ 분포
  • $\mathcal{V}_k(t)$: teacher 기준 상위 $k=50$ 토큰 집합 (kd_topk=50)

코드 대응:

topk_idx = t_logits.topk(50, dim=-1).indices     # teacher top-50
s_logits  = s_logits.gather(-1, topk_idx)         # student도 동일 위치
t_logits  = t_logits.gather(-1, topk_idx)

s_logp = log_softmax(s_logits / T_kd)
t_logp = log_softmax(t_logits / T_kd)

kl = (s_logp.exp() * (s_logp - t_logp)).sum(dim=-1)   # KL per token
kd_loss = (kl * gen_mask).sum() / gen_mask.sum()

Reverse KL: student이 teacher에 비해 mode-seeking. Forward KL보다 분포가 날카로워지는 경향.

3-4. 최종 loss

$$\mathcal{L}(w) \;=\; \mathcal{L}_\text{KD} + \lambda_\text{NTP} \cdot \mathcal{L}_\text{NTP}$$

현재: $\lambda_\text{NTP} = 0$ → $\mathcal{L}(w) = \mathcal{L}_\text{KD}$

4. 전체 알고리즘 흐름

초기화: $z^0 = \text{Proj}(w_0)$, $u^0 = 0$

for $t = 1, \ldots, T$:

  1. $y \sim \pi_\theta(\cdot|x)$  (on-policy generation, no grad)
  2. $g_t = \nabla_w \mathcal{L}_\text{KD}(w; x, y) + \lambda(w - z + u)$
  3. $w_{t+1} = w_t - \text{Adam}(g_t)$
  4. if $t \bmod 32 = 0$:
    • $z \leftarrow \text{Proj}_{\mathcal{S}_s}(w + u)$
    • $u \leftarrow u + (w - z)$

최종: $w \leftarrow \text{Proj}(w)$ (hard sparsification)

5. Lambda 문제

현재 설정: lr=1e-2, lmda=5e-5

원래 ELSA 논문 세팅: lr=1e-1, lmda=5e-5

→ lr을 10배 낮췄는데 lmda는 그대로.

Adam step의 effective scale은 lr에 비례. proximal term $\lambda(w-z+u)$는 gradient에 절대값으로 더해지므로, lr이 낮을수록 Adam step 대비 penalty의 상대적 영향이 커짐.

즉, task gradient $\nabla \mathcal{L}_\text{KD}$가 줄어드는 상황에서 penalty $\lambda(w-z+u)$는 그대로 → ADMM sparsity 제약이 상대적으로 너무 강하게 작용할 수 있음.

설정lrlmdalmda/lr 비율
논문 세팅1e-15e-55e-4
현재 스크립트1e-25e-55e-3 (10배)
비율 맞춘 경우1e-25e-65e-4

6. 속도 문제

현재: BS=1, grad_accum=8 → training_step()이 8번 호출 → generate() 8번 순차 실행

→ 8 × 73s = 584s/step (~9.7분), 1024 steps × 9.7분 = ETA 166시간

Fix: BS=8, grad_accum=1로 변경하면 generate()를 1번만 호출

→ 약 73s/step, 1024 steps = 약 20시간 (8× 개선)
